Causal Sense

AI-Native Causal Inference Engine

Outcome

The idea is hardened. Causal Sense has a single, defensible identity: an AI-native causal inference engine that answers "what actually caused this?" — not a generic data science assistant. All four teammate features are non-redundant and load-bearing in one coherent pipeline. The product is ready for PRD and architecture planning.

Locked Decisions

  • Single shared-tools agent architecture — eliminates redundancy, all steps share tools
  • Market research runs upstream (informs DAG construction), not downstream as post-hoc justification
  • Regression/prediction is counterfactual: what-if simulator uses the causal graph, not plain regression
  • Causal inference is the core product identity — all features serve one answer
  • 10-day timeline — full 7-step pipeline is feasible to build and demo

Rejected Options

  • 4 separate per-teammate agents — creates redundant tools and prevents cross-feature tool sharing
  • Market research as downstream explainer — post-hoc rationalization and hallucination risk
  • Generic DS assistant framing — weak on Innovation criterion (20% of score)

Surviving Weak Points

  • LLMs can hallucinate DAG edges — expert validation canvas must be mandatory, not optional
  • AI tool use (25% of score, highest weight) — Claude's role in each step must be explicitly articulated in the demo and docs
  • 7-step pipeline scope — recommend anchoring the demo to one end-to-end dataset (e.g. gold prices) to keep it tight
